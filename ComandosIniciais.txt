Iniciar projeto Angular

npm i -g @angular/cli (instalar o Angular)

ng new nomedoprojeto --minimal (projeto simples com o mínimo necessário)

***DICAS INICIAIS***
"@schematics/angular:component": {
"inlineTemplate": true,   ***trocar para false "inlineTemplate": false,
"inlineStyle": true,      ***trocar para false  "inlineStyle": false,
"skipTests": true

***Porque? -> Desta forma os arquivos HTML (inlineTemplate), CSS (inlineStyle) estarão em arquivos separados... Caso contrário ficariam em um único arquivo por padrão por termos uma aplicação mínima (---minimal)...***

*O main.ts aponta e carrega os arquivos iniciais que estão na pasta app

Inicia-se assim: main.ts (pasta src) -> app.module "bootstrap: [app.component]" (pasta app) -> app.
component

***INSTALAR NOVOS RECURSOS***

ng add @angular/nomedorecurso

No caso do material (ng add @angular/material) virão perguntas...
1ª Template (escolher - ver na url o exemplo)
2ª Tipografia (sim ou não)
3ª Imagens (sim ou não)

***Para gerar componentes***
ng generate component caminho/algumacoisa (pode ser trocado por g c - generate component)
ex: ng g c components/template/header

***Criar Diretivas***
ng g d directives/red (pasta/diretiva) - diretiva de atributos, como estilos
ng g d directives/for (pasta/diretiva) - diretiva estrutural, como adicionar elementos na página (DOM) ou componente

***Bidings***
biding de evento como um evento de cliques. Usa-se entre parenteses (click)

***Services***
exemplo: ng g s components/product/product
Os services estão relacionados com códigos e outros componentes como conexões HTTP; por convenção e não regra, os services devem ser os mais enxutos possíveis e não devem se relacionar com objetivos de apresentar nada em tela
Não aparecem em app.module nas declarations pois estão reservados componentes e diretivas, caso for necessário expor os services estes devem estar em providers (arquivo app.module)
